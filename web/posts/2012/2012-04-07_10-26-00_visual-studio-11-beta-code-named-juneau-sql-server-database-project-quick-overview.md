<p>I think one of the most awesome features of Visual Studio 11 beta is SQL Server Database Project (code-named "Juneau"). It is simple, robust and makes it really easy for us to check the database code into our source control system.</p>
<blockquote>
<p>I hear you yelling at me "Dude, ever heard about <a target="_blank" href="http://en.wikipedia.org/wiki/NoSQL" title="http://en.wikipedia.org/wiki/NoSQL">NoSQL</a>? <a target="_blank" href="http://www.mongodb.org/" title="http://www.mongodb.org/">MongoDB</a>? <a target="_blank" href="http://ravendb.net/" title="http://ravendb.net/">RavenDB</a>?". Yeah, I am aware of those technologies and I think you can achieve lots of things that you can do with a relational database system with a NoSQL database system. On the other hand, there are quite a lot of other things that you cannot do with a NoSQL database system. At least, with the comfort of SQL. What? Did you say "Comfort? Who favors comfort over performance?"? I would like to remind you that all <a target="_blank" href="http://stackexchange.com/" title="http://stackexchange.com/">Stack Exchange</a> family sites run on SQL Server.</p>
<p>To sum up, SQL Server will not go anywhere at all. It will always be around and we will keep using it. So, I&lsquo;d pay attention to this post <img src="http://www.tugberkugurlu.com/Content/Images/UploadedByAuthors/wlw/Visual-Studio-11-Beta-Database-Project_B3E6/wlEmoticon-smile.png" alt="Smile" class="wlEmoticon wlEmoticon-smile" style="border-style: none;" /></p>
</blockquote>
<p>This project template carries lots of features inside it and despite the fact that it is so easy to get started with it, you might drown inside its options very quickly. Honest to say, I am not so experienced with this feature but I try to use this on some personal projects for now as much as I can.</p>
<p>Let&rsquo;s first have a look at Visual Studio 2010 because if you play with SQL Server project templates with VS 2010, you will be in panic because lots of them are now gone.</p>
<p><a href="http://www.tugberkugurlu.com/Content/Images/UploadedByAuthors/wlw/Visual-Studio-11-Beta-Database-Project_B3E6/image.png"><img height="394" width="644" src="http://www.tugberkugurlu.com/Content/Images/UploadedByAuthors/wlw/Visual-Studio-11-Beta-Database-Project_B3E6/image_thumb.png" alt="image" border="0" title="image" style="background-image: none; padding-left: 0px; padding-right: 0px; display: inline; padding-top: 0px; border: 0px;" /></a></p>
<p>Ok, then. What do we do now? Well, it is simple: they are now all in one project template. From database projects to SQL Server CLR assembly projects, SQL Server Database Project template can do all of them (I have never tired creating a SQL Server CLR project with VS 11 Database Project template so far. So, I might not be entirely correct here).</p>
<p>Let&rsquo;s come back to VS 11 beta. Creating a SQL Server Database Project is very straight forward. File &gt; New &gt; Project and navigate to SQL Server language tab or search against "SQL Server". SQL Server Database Project option will be available.</p>
<p><a href="http://www.tugberkugurlu.com/Content/Images/UploadedByAuthors/wlw/Visual-Studio-11-Beta-Database-Project_B3E6/image_3.png"><img height="394" width="644" src="http://www.tugberkugurlu.com/Content/Images/UploadedByAuthors/wlw/Visual-Studio-11-Beta-Database-Project_B3E6/image_thumb_3.png" alt="image" border="0" title="image" style="background-image: none; padding-left: 0px; padding-right: 0px; display: inline; padding-top: 0px; border: 0px;" /></a></p>
<p>When we create the project, we get not many things out of the box which is nice.</p>
<p><a href="http://www.tugberkugurlu.com/Content/Images/UploadedByAuthors/wlw/Visual-Studio-11-Beta-Database-Project_B3E6/image_4.png"><img height="273" width="304" src="http://www.tugberkugurlu.com/Content/Images/UploadedByAuthors/wlw/Visual-Studio-11-Beta-Database-Project_B3E6/image_thumb_4.png" alt="image" border="0" title="image" style="background-image: none; padding-left: 0px; padding-right: 0px; display: inline; padding-top: 0px; border: 0px;" /></a></p>
<p>From this point, you can either start a brand new project or import your existing database.</p>
<p>When you double-click on the <strong>Properties</strong>, we will get properties window which we can set lots of configuration about the database, build process and so on.</p>
<p><a href="http://www.tugberkugurlu.com/Content/Images/UploadedByAuthors/wlw/Visual-Studio-11-Beta-Database-Project_B3E6/image_5.png"><img height="366" width="644" src="http://www.tugberkugurlu.com/Content/Images/UploadedByAuthors/wlw/Visual-Studio-11-Beta-Database-Project_B3E6/image_thumb_5.png" alt="image" border="0" title="image" style="background-image: none; padding-left: 0px; padding-right: 0px; display: inline; padding-top: 0px; border: 0px;" /></a></p>
<p>When you are working on a project, make sure you set the target properly as shown below.</p>
<p><a href="http://www.tugberkugurlu.com/Content/Images/UploadedByAuthors/wlw/Visual-Studio-11-Beta-Database-Project_B3E6/image_6.png"><img height="364" width="644" src="http://www.tugberkugurlu.com/Content/Images/UploadedByAuthors/wlw/Visual-Studio-11-Beta-Database-Project_B3E6/image_thumb_6.png" alt="image" border="0" title="image" style="background-image: none; padding-left: 0px; padding-right: 0px; display: inline; padding-top: 0px; border: 0px;" /></a></p>
<p>While we are working on our database project, we will be creating files for Tables, User Defined Functions, Stored Procedures, etc. Don&rsquo;t worry about where you put them. VS is smart about your files and it orders them properly on your deployment. This is the experience I have had so far.</p>
<p>As usual, we can add new items via Add New Item dialog box and Database Project ships with several item templates.</p>
<p><a href="http://www.tugberkugurlu.com/Content/Images/UploadedByAuthors/wlw/Visual-Studio-11-Beta-Database-Project_B3E6/image_7.png"><img height="150" width="244" src="http://www.tugberkugurlu.com/Content/Images/UploadedByAuthors/wlw/Visual-Studio-11-Beta-Database-Project_B3E6/image_thumb_7.png" alt="image" border="0" title="image" style="background-image: none; padding-left: 0px; padding-right: 0px; display: inline; padding-top: 0px; border: 0px;" /></a><a href="http://www.tugberkugurlu.com/Content/Images/UploadedByAuthors/wlw/Visual-Studio-11-Beta-Database-Project_B3E6/image_8.png"><img height="150" width="244" src="http://www.tugberkugurlu.com/Content/Images/UploadedByAuthors/wlw/Visual-Studio-11-Beta-Database-Project_B3E6/image_thumb_8.png" alt="image" border="0" title="image" style="background-image: none; padding-left: 0px; padding-right: 0px; display: inline; padding-top: 0px; border: 0px;" /></a><a href="http://www.tugberkugurlu.com/Content/Images/UploadedByAuthors/wlw/Visual-Studio-11-Beta-Database-Project_B3E6/image_9.png"><img height="150" width="244" src="http://www.tugberkugurlu.com/Content/Images/UploadedByAuthors/wlw/Visual-Studio-11-Beta-Database-Project_B3E6/image_thumb_9.png" alt="image" border="0" title="image" style="background-image: none; padding-left: 0px; padding-right: 0px; display: inline; padding-top: 0px; border: 0px;" /></a></p>
<p>User experience is also incredibly high. For example, you can create your tables through design surface or directly with T-SQL code. No matter which one you pick, VS will always keep those in sync.</p>
<p>As you can see below, we have a file named Products.sql for <strong>Products</strong> table.</p>
<p><a href="http://www.tugberkugurlu.com/Content/Images/UploadedByAuthors/wlw/Visual-Studio-11-Beta-Database-Project_B3E6/image_10.png"><img height="395" width="644" src="http://www.tugberkugurlu.com/Content/Images/UploadedByAuthors/wlw/Visual-Studio-11-Beta-Database-Project_B3E6/image_thumb_10.png" alt="image" border="0" title="image" style="background-image: none; padding-left: 0px; padding-right: 0px; display: inline; padding-top: 0px; border: 0px;" /></a></p>
<p>Let&rsquo;s create a foreign key constraint for <strong>CategoryId </strong>column by referencing a table which does not exist yet.</p>
<p><a href="http://www.tugberkugurlu.com/Content/Images/UploadedByAuthors/wlw/Visual-Studio-11-Beta-Database-Project_B3E6/image_11.png"><img height="219" width="644" src="http://www.tugberkugurlu.com/Content/Images/UploadedByAuthors/wlw/Visual-Studio-11-Beta-Database-Project_B3E6/image_thumb_11.png" alt="image" border="0" title="image" style="background-image: none; padding-left: 0px; padding-right: 0px; display: inline; padding-top: 0px; border: 0px;" /></a></p>
<p>Save the file and build the solution. Now, we are getting errors since the <strong>Categories</strong> table does not exist.</p>
<p><a href="http://www.tugberkugurlu.com/Content/Images/UploadedByAuthors/wlw/Visual-Studio-11-Beta-Database-Project_B3E6/image_12.png"><img height="212" width="644" src="http://www.tugberkugurlu.com/Content/Images/UploadedByAuthors/wlw/Visual-Studio-11-Beta-Database-Project_B3E6/image_thumb_12.png" alt="image" border="0" title="image" style="background-image: none; padding-left: 0px; padding-right: 0px; display: inline; padding-top: 0px; border: 0px;" /></a></p>
<p>When we add the necessary <strong>Categories</strong> table and its columns, the build process should run smoothly without any errors.</p>
<p>Let&rsquo;s now open up the <strong>Products.sql</strong> table file again. Did you notice the right side of the design surface? Yes, it really knows what is going on outside of this file and it keeps the information in sync. As soon as you make a chance anywhere inside your project, proper places get updated.</p>
<p><a href="http://www.tugberkugurlu.com/Content/Images/UploadedByAuthors/wlw/Visual-Studio-11-Beta-Database-Project_B3E6/image_13.png"><img height="364" width="644" src="http://www.tugberkugurlu.com/Content/Images/UploadedByAuthors/wlw/Visual-Studio-11-Beta-Database-Project_B3E6/image_thumb_13.png" alt="image" border="0" title="image" style="background-image: none; padding-left: 0px; padding-right: 0px; display: inline; padding-top: 0px; border: 0px;" /></a></p>
<p>You can also create script files which will be executed at particular point of your deployment. When you go Add New Item dialog box and navigate to User Scripts tab, you will see some options there. Let&rsquo;s add <strong>Post-Deployment Script</strong> file to our project.</p>
<p><a href="http://www.tugberkugurlu.com/Content/Images/UploadedByAuthors/wlw/Visual-Studio-11-Beta-Database-Project_B3E6/image_14.png"><img height="394" width="644" src="http://www.tugberkugurlu.com/Content/Images/UploadedByAuthors/wlw/Visual-Studio-11-Beta-Database-Project_B3E6/image_thumb_14.png" alt="image" border="0" title="image" style="background-image: none; padding-left: 0px; padding-right: 0px; display: inline; padding-top: 0px; border: 0px;" /></a></p>
<p>This file is nothing but a regular file except for the fact that its build action has been set to <strong>PostDeploy</strong>.</p>
