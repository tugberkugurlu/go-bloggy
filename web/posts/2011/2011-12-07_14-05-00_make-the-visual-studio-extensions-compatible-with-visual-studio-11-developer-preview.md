<p><a href="https://www.tugberkugurlu.com/Content/images/Uploadedbyauthors/wlw/da94e67e689d_F189/works-on-my-machine-seal-of-approval.png"><img style="background-image: none; margin: 0px 15px 10px 0px; padding-left: 0px; padding-right: 0px; display: inline; float: left; padding-top: 0px; border: 0px;" title="works-on-my-machine-seal-of-approval" border="0" alt="works-on-my-machine-seal-of-approval" align="left" src="https://www.tugberkugurlu.com/Content/images/Uploadedbyauthors/wlw/da94e67e689d_F189/works-on-my-machine-seal-of-approval_thumb.png" width="200" height="193" /></a>Do you use <a title="http://msdn.microsoft.com/en-us/vstudio/hh127353" href="http://msdn.microsoft.com/en-us/vstudio/hh127353" target="_blank">Visual Studio 11 Developer Preview</a> and miss your extensions? Well, I do! Especially, <a title="http://visualstudiogallery.msdn.microsoft.com/7812b04c-db36-4817-8a84-e73c452410a2" href="http://visualstudiogallery.msdn.microsoft.com/7812b04c-db36-4817-8a84-e73c452410a2" target="_blank">ADO.NET C# DbContext Generator</a>.</p>
<p>As you know, for now Visual Studio extensions are not compatible with Visual Studio 11 Developer Preview.</p>
<p><a href="https://www.tugberkugurlu.com/Content/images/Uploadedbyauthors/wlw/da94e67e689d_F189/image.png"><img style="background-image: none; padding-left: 0px; padding-right: 0px; display: inline; padding-top: 0px; border: 0px;" title="image" border="0" alt="image" src="https://www.tugberkugurlu.com/Content/images/Uploadedbyauthors/wlw/da94e67e689d_F189/image_thumb.png" width="244" height="97" /></a></p>
<p>This is normal because it is <strong>Developer Preview</strong>. Even, I opened up a thread for that on <a title="http://stackoverflow.com" href="http://stackoverflow.com" target="_blank">Stackoverflow</a>: <a href="http://stackoverflow.com/questions/7726121/visual-studio-extensions-and-visual-studio-11-dev-preview" target="_blank">Visual Studio Extensions and Visual Studio 11 Dev. Preview</a> and <a title="http://stackoverflow.com/users/629083/betty" href="http://stackoverflow.com/users/629083/betty" target="_blank">@betty</a> replied to my post after a long time with <a title="http://stackoverflow.com/a/8335636/463785" href="http://stackoverflow.com/a/8335636/463785" target="_blank">an amazing answer</a>. I never developed a Visual Studio extension before and I am not familiar with the drill so I thought it is not possible to tweak the extensions so that they can be compatible with Visual Studio 11 Developer Preview but it turns out that it is as easy as pie. How so? Here are the steps:</p>
<p>Oww, by the way, this thing <strong>works on my machine </strong>and I am not guarantee that it will work on yours as well. Even worse, it can completely screw your machine up. So, take the risk here and don&rsquo;t swear to me.</p>
<p>I also assume that the <a title="http://visualstudiogallery.msdn.microsoft.com/7812b04c-db36-4817-8a84-e73c452410a2" href="http://visualstudiogallery.msdn.microsoft.com/7812b04c-db36-4817-8a84-e73c452410a2" target="_blank">ADO.NET C# DbContext Generator</a> does not integrate with Visual Studio that much so there was no big compatibility problems but I won&rsquo;t try this for the extensions like <a title="http://visualstudiogallery.msdn.microsoft.com/d491911d-97f3-4cf6-87b0-6a2882120acf?SRC=Home" href="http://visualstudiogallery.msdn.microsoft.com/d491911d-97f3-4cf6-87b0-6a2882120acf?SRC=Home" target="_blank">VSCommands 2010</a>, <a title="http://visualstudiogallery.msdn.microsoft.com/6ed4c78f-a23e-49ad-b5fd-369af0c2107f?SRC=Home" href="http://visualstudiogallery.msdn.microsoft.com/6ed4c78f-a23e-49ad-b5fd-369af0c2107f?SRC=Home" target="_blank">Web Essentials</a>, etc. If you try, please let me know how it went. Anyway, here is the drill:</p>
<p>First of all, download the extension from <a title="http://visualstudiogallery.msdn.microsoft.com" href="http://visualstudiogallery.msdn.microsoft.com" target="_blank">Visual Studio Gallery</a>.</p>
<p><a href="https://www.tugberkugurlu.com/Content/images/Uploadedbyauthors/wlw/da94e67e689d_F189/image_3.png"><img style="background-image: none; padding-left: 0px; padding-right: 0px; display: inline; padding-top: 0px; border: 0px;" title="image" border="0" alt="image" src="https://www.tugberkugurlu.com/Content/images/Uploadedbyauthors/wlw/da94e67e689d_F189/image_thumb_3.png" width="644" height="396" /></a></p>
<p>I am not sure if you are aware of it but vsix files are actually zip files just like <a title="http://nuget.org" href="http://nuget.org" target="_blank">NuGet</a> package files (nupkg) and MS Word files (docx). So, change the extension of the file we have downloaded from vsix to zip:</p>
<p><a href="https://www.tugberkugurlu.com/Content/images/Uploadedbyauthors/wlw/da94e67e689d_F189/image_4.png"><img style="background-image: none; margin: 0px 15px 0px 0px; padding-left: 0px; padding-right: 0px; display: inline; padding-top: 0px; border: 0px;" title="image" border="0" alt="image" src="https://www.tugberkugurlu.com/Content/images/Uploadedbyauthors/wlw/da94e67e689d_F189/image_thumb_4.png" width="244" height="150" /></a>&nbsp;<a href="https://www.tugberkugurlu.com/Content/images/Uploadedbyauthors/wlw/da94e67e689d_F189/image_5.png"><img style="background-image: none; margin: 0px; padding-left: 0px; padding-right: 0px; display: inline; padding-top: 0px; border: 0px;" title="image" border="0" alt="image" src="https://www.tugberkugurlu.com/Content/images/Uploadedbyauthors/wlw/da94e67e689d_F189/image_thumb_5.png" width="244" height="150" /></a></p>
<p>Unzip the zip file and you will see a file called <strong>extension.vsixmanifest </strong>under root folder. No, it is not a zip file, too <img style="border-style: none;" class="wlEmoticon wlEmoticon-openmouthedsmile" alt="Open-mouthed smile" src="https://www.tugberkugurlu.com/Content/images/Uploadedbyauthors/wlw/da94e67e689d_F189/wlEmoticon-openmouthedsmile.png" /> It is an xml file though. Open it up with a text editor and find the node called <strong>SupportedProducts</strong>:</p>
<p><a href="https://www.tugberkugurlu.com/Content/images/Uploadedbyauthors/wlw/da94e67e689d_F189/image_6.png"><img style="background-image: none; padding-left: 0px; padding-right: 0px; display: inline; padding-top: 0px; border: 0px;" title="image" border="0" alt="image" src="https://www.tugberkugurlu.com/Content/images/Uploadedbyauthors/wlw/da94e67e689d_F189/image_thumb_6.png" width="644" height="357" /></a></p>
<p>Just change the value of version attribute of VisualStudio node to <strong>11.0 </strong>and save it.</p>
<p><a href="https://www.tugberkugurlu.com/Content/images/Uploadedbyauthors/wlw/da94e67e689d_F189/image_7.png"><img style="background-image: none; padding-left: 0px; padding-right: 0px; display: inline; padding-top: 0px; border: 0px;" title="image" border="0" alt="image" src="https://www.tugberkugurlu.com/Content/images/Uploadedbyauthors/wlw/da94e67e689d_F189/image_thumb_7.png" width="644" height="357" /></a></p>
<p>This is actually enough for the extension compatibility standpoint but <strong>ADO.NET C# DbContext Generator</strong> extension has some dependencies which are specific to Visual Studio 2010 (not sure on that though). So,we have one more step to go as well.</p>
<p>Navigate to <strong>/T/code/</strong> folder and you will see a zip file there:</p>
<p><a href="https://www.tugberkugurlu.com/Content/images/Uploadedbyauthors/wlw/da94e67e689d_F189/image_8.png"><img style="background-image: none; padding-left: 0px; padding-right: 0px; display: inline; padding-top: 0px; border: 0px;" title="image" border="0" alt="image" src="https://www.tugberkugurlu.com/Content/images/Uploadedbyauthors/wlw/da94e67e689d_F189/image_thumb_8.png" width="644" height="393" /></a></p>
<p>Unzip it:</p>
<p><a href="https://www.tugberkugurlu.com/Content/images/Uploadedbyauthors/wlw/da94e67e689d_F189/image_9.png"><img style="background-image: none; padding-left: 0px; padding-right: 0px; display: inline; padding-top: 0px; border: 0px;" title="image" border="0" alt="image" src="https://www.tugberkugurlu.com/Content/images/Uploadedbyauthors/wlw/da94e67e689d_F189/image_thumb_9.png" width="644" height="393" /></a></p>

<p>Don&rsquo;t ask me why because I have no clue. I assumed that it was a versioning issue and it works this way. Remember, this is developer preview. Not even alpha or beta so don&rsquo;t look for logic that much.</p>
<p>Save the file and zip it up again.</p>
<p><a href="https://www.tugberkugurlu.com/Content/images/Uploadedbyauthors/wlw/da94e67e689d_F189/image_10.png"><img style="background-image: none; padding-left: 0px; padding-right: 0px; display: inline; padding-top: 0px; border: 0px;" title="image" border="0" alt="image" src="https://www.tugberkugurlu.com/Content/images/Uploadedbyauthors/wlw/da94e67e689d_F189/image_thumb_10.png" width="644" height="368" /></a>,</p>
<p>Don&rsquo;t forget to delete other files. You just need that zip file inside the <strong>code</strong> folder.</p>
<p>Lastly, go under the root and zip all the files:</p>
<p><a href="https://www.tugberkugurlu.com/Content/images/Uploadedbyauthors/wlw/da94e67e689d_F189/image_11.png"><img style="background-image: none; padding-left: 0px; padding-right: 0px; display: inline; padding-top: 0px; border: 0px;" title="image" border="0" alt="image" src="https://www.tugberkugurlu.com/Content/images/Uploadedbyauthors/wlw/da94e67e689d_F189/image_thumb_11.png" width="644" height="366" /></a></p>

<p><a href="https://www.tugberkugurlu.com/Content/images/Uploadedbyauthors/wlw/da94e67e689d_F189/image_13.png"><img style="background-image: none; margin: 0px 15px 0px 0px; padding-left: 0px; padding-right: 0px; display: inline; padding-top: 0px; border: 0px;" title="image" border="0" alt="image" src="https://www.tugberkugurlu.com/Content/images/Uploadedbyauthors/wlw/da94e67e689d_F189/image_thumb_13.png" width="244" height="150" /></a><a href="https://www.tugberkugurlu.com/Content/images/Uploadedbyauthors/wlw/da94e67e689d_F189/image_14.png"><img style="background-image: none; margin: 0px; padding-left: 0px; padding-right: 0px; display: inline; padding-top: 0px; border: 0px;" title="image" border="0" alt="image" src="https://www.tugberkugurlu.com/Content/images/Uploadedbyauthors/wlw/da94e67e689d_F189/image_thumb_14.png" width="244" height="150" /></a></p>
